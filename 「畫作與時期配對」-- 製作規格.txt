「畫作與時期配對」-- 製作規格

頁面規格說明書
一、基本結構
單一 HTML 檔案，CSS/JS 全部內寫，不依賴外部框架
檔名：index.html
二、資料設定
在 JS 中建兩個陣列：

1. 分類陣列（categories）

[
  { id: "dutch",     name: "荷蘭時期" },
  { id: "paris",     name: "巴黎時期" },
  { id: "arles",     name: "阿爾時期" },
  { id: "saint-auv", name: "聖雷米與奧維時期" }
]
id：程式內部識別用（對應到每張圖的 period）
name：畫面上顯示的分類名稱
2. 畫作陣列（paintings）

[
  { img: "圖片路徑", period: "對應的分類id", w: 圖片寬, h: 圖片高 },
  ...
]
img：圖片的相對路徑
period：此畫作所屬的分類 id（一對一或一對多皆可，數量不拘）
w、h：圖片原始像素尺寸，用於動態計算卡片比例
三、圖片處理
圖片統一放在獨立資料夾中（如 各時期畫作/）
HTML 檔案若在子資料夾，路徑需用 ../ 往上一層找
卡片寬度固定（如 120px），高度依 w:h 比例自動計算
直向圖要保留原始比例，不裁切不變形
圖片不顯示名稱，純圖片卡片
四、遊戲機制
功能	說明
拖曳配對	卡片可拖曳到任意分類欄，不論對錯都接受
不即時判斷	拖入時不顯示對錯，留到提交時一起判定
提交按鈕	全部放好才能提交，缺件會提醒
答對標記	綠色邊框
答錯標記	紅色邊框
重新作答	答錯時出現，將所有卡片倒回原池、清除標記
再玩一次	答對時出現，重新洗牌開始新一局
手機觸控	需支援 touchstart/touchmove/touchend 拖曳操作
五、視覺風格（黏土風格）
項目	設定
字體	辰宇落雁體（@font-face 從 GitHub CDN 載入）
背景	大地色 #e8c9a0，搭配 radial-gradient 光暈
區塊	漸層背景 + 多層 box-shadow（外陰影 + 內高光）模擬立體黏土感
圓角	大圓角（14px~26px）
標題欄	深棕色漸層底 + 白字
提交按鈕	紅色漸層
重玩按鈕	棕色漸層
六、特效（答對時）
從畫面底部往上噴出 80 朵向日葵
集中在畫面中間 70% 寬度
初速度向上，搭配重力自然落下
有旋轉、微幅搖擺動畫
約 240 幀後逐漸淡出消失
使用 Canvas 繪製（SVG 圖層會擋住操作，用 pointer-events: none）
七、響應式設計
桌面：4 欄（4 個分類並排）
平板（<800px）：2 欄
手機（<500px）：1 欄
